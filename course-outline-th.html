The AI orchestrator mindset — why we're here
ก่อนที่จะแตะเครื่องมือใดๆ ให้สร้าง mental model ที่เป็นรากฐานของหลักสูตรทั้งหมดนี้ก่อน งาน IT แบบดั้งเดิมเป็นแบบ imperative: คุณเขียนคำสั่งทีละขั้นอย่างชัดเจน — scripts, automations, workflows — และเครื่องทำตามคำสั่งนั้นตรงๆ Claude Code เพิ่ม layer ใหม่: intelligence ที่สามารถ reason จาก intent ของคุณ คิดหาขั้นตอนเอง และลงมือ execute ได้ หน้าที่ของคุณเปลี่ยนจากการเขียนทุกคำสั่ง ไปเป็นการ สั่งการ agent แล้ว review และอนุมัติผลลัพธ์ นั่นคือสิ่งที่ orchestrator ทำ

แนวคิดหลักคือ incumbent + intelligence — ทักษะและความรู้เดิม + ปัญญาของ AI ความรู้ domain ของคุณ — ว่า tickets ไหลอย่างไร board ที่ healthy ควรเป็นอย่างไร Jira automation ควรทำอะไร — ไม่ได้ล้าสมัย แต่กลายเป็น input ที่มีคุณค่าที่สุดในระบบ Claude Code ทำหน้าที่ด้านความเร็วในการ execute และจดจำ syntax ส่วนคุณทำหน้าที่ตัดสินใจว่าอะไรควรสร้างและผลลัพธ์ถูกต้องหรือไม่
ประโยคสำหรับห้องเรียน: "คุณไม่ได้มาที่นี่เพื่อเรียนเขียนโค้ด คุณมาเพื่อเรียนวิธีสั่งการสิ่งที่เขียนโค้ดได้ — และรู้ว่าเมื่อไหรควรเชื่อใจมัน เมื่อไหรควรแก้ทิศทาง และเมื่อไหรควรเริ่มใหม่" ทุกนิสัยในหลักสูตรนี้ — /plan ก่อน build, CLAUDE.md เพื่อ set context, review diff ก่อน accept — ล้วนเป็นนิสัยของ orchestrator ไม่ใช่นิสัยของ coder

คุ้นเคยมากที่สุด
Claude.ai
อินเทอร์เฟซ chat — ที่ที่ทีมของคุณคุ้นเคยอยู่แล้ว ใช้สำหรับถาม-ตอบทั่วไป ร่างข้อความ และสรุปเนื้อหา
Connectors
การเชื่อมต่อแบบ one-click ที่ built-in ใน Claude.ai — Atlassian, Google Drive, Gmail, Calendar ไม่ต้องตั้งค่าอะไรนอกจากการ link บัญชี Connectors อยู่ที่นี่ ไม่ใช่ใน Claude Code
จุดมุ่งหมายของหลักสูตรนี้
Claude Code
AI coding assistant แบบ terminal-based อ่านไฟล์โปรเจกต์จริงของคุณ และเขียน รัน และแก้ไขโค้ด ใช้งานผ่าน terminal หรือ VS Code extension
MCP (Model Context Protocol)
วิธีที่ Claude Code เชื่อมต่อกับเครื่องมือภายนอก เช่น Jira, GitHub, databases พร้อม programmatic control เต็มรูปแบบ แนะนำใน Session 3
สำหรับผู้ที่ไม่ใช่ developer
Cowork ✦ new
Desktop app สำหรับผู้ที่ไม่ใช่ developer ช่วย automate การจัดการไฟล์และงานโดยไม่ต้องใช้ terminal เป็นสะพานเชื่อมระหว่าง Claude.ai และ Claude Code
เหมาะสำหรับผู้เข้าร่วมที่รู้สึกกังวลกับ terminal — Cowork คือทางเลือกที่เข้าถึงได้ทันที
เคล็ดลับการสอน: วาด spectrum จากซ้ายไปขวา — Claude.ai (ใช้งานง่ายที่สุด ควบคุมน้อยที่สุด) → Cowork (no-code desktop) → Claude Code (ทรงพลังที่สุด ใช้ terminal) ถามว่าตอนนี้ผู้เข้าร่วมอยู่ตรงไหนและอยากไปถึงจุดไหน ประโยคสำหรับชั้นเรียน: "Connectors คือสิ่งที่ Connectors เป็นให้ Claude.ai เหมือนกับที่ MCP เป็นให้ Claude Code — concept เดียวกัน แต่ power level ต่างกัน"

Install Claude Code via terminal
Native installer — ไม่ต้องติดตั้ง Node.js รัน install พร้อมกันสำหรับทั้งสอง OS macOS: คำสั่ง curl เดียวใน Terminal Windows: คำสั่งเดียวใน PowerShell ยืนยันตัวตนด้วยบัญชี Anthropic ที่มีอยู่แล้ว ตรวจสอบด้วย claude --version
⚠ Windows: ต้องติดตั้ง Git for Windows ก่อน (ดู pre-session checklist) ใช้ integrated terminal ของ VS Code เป็น backup หาก PowerShell มีปัญหา

How Claude Code reads your project
อธิบาย context: Claude Code เห็นไฟล์ทั้งหมดในโฟลเดอร์ที่เปิดใน VS Code แนะนำ CLAUDE.md — ไฟล์ plain text ที่คุณเขียนครั้งเดียวเพื่อบอก Claude เกี่ยวกับโปรเจกต์ conventions และ preferences ของทีม Demo: สร้าง CLAUDE.md ที่อธิบายโครงสร้าง Atlassian project ทั่วไป แสดงให้เห็นว่ามันช่วยลดการอธิบายซ้ำๆ ใน prompt ได้อย่างไร นี่คือช่วงเวลา "ตั้งค่าครั้งเดียว ได้ประโยชน์ตลอด"
ใน VS Code, CLAUDE.md จะปรากฏเป็นไฟล์ปกติใน explorer — ผู้เข้าร่วมสามารถดูและแก้ไขได้โดยไม่ต้องออกจาก workspace

concept
/plan mode — think before you build
ก่อนที่จะขอให้ Claude Code เขียนหรือแก้ไขไฟล์ใดๆ ให้ใช้ /plan เพื่อขอให้อธิบาย approach ก่อน Claude จะบอกว่าตั้งใจจะทำอะไร — ไฟล์ไหนที่จะสร้างหรือแก้ไข logic อะไรที่จะใช้ — โดยไม่เปลี่ยนแปลงอะไรเลย คุณ review แผน แก้ไขทิศทางถ้าจำเป็น แล้วค่อยดำเนินการต่อ สำหรับผู้ที่ไม่ใช่ developer สิ่งนี้มีคุณค่าเป็นพิเศษ: สร้างความเชื่อมั่น จับความเข้าใจผิดได้ตั้งแต่ต้น และทำให้ iteration loop รู้สึกโปร่งใสแทนที่จะรู้สึกเหมือน black box ประโยคสำหรับชั้นเรียน: "Plan ก่อน แล้วค่อย build"
/plan ยังเป็น token-efficiency tool อีกด้วย — การจับทิศทางผิดก่อนที่ Claude จะเขียนโค้ด 100 บรรทัด ประหยัด token ได้มากกว่า /compact มาก นำนิสัยนี้ติดตัวเข้าสู่ workshop ที่ตามมา

workshop
Build: project health snapshot generator
ให้ Claude Code ดู CSV export ของ open tickets จาก Jira หรือ monday.com และขอให้สร้าง Python script ที่ผลิต HTML report หน้าเดียวที่สะอาด แสดง: สถานะ breakdown ของ tickets (open / in progress / blocked / done), overdue items ที่ highlight ตามอายุ และสรุป workload ของแต่ละ assignee นี่คือ report ที่ทุกทีม IT ทำด้วยมือซ้ำแล้วซ้ำเล่า — output จาก workshop สามารถแชร์ได้เช้าวันต่อมา

เริ่มด้วย /plan: ก่อนที่จะเขียนโค้ดแม้แต่บรรทัดเดียว ขอให้ Claude Code วางแผนว่าจะจัดโครงสร้าง report อย่างไร review แผนร่วมกันและแก้ทิศทางถ้าจำเป็น — "เพิ่ม section สำหรับ tickets ที่ไม่มี assignee" หรือ "เรียง overdue จากเก่าที่สุด" แล้วค่อยดำเนินการ ผู้สอนสาธิต loop ทั้งหมดก่อน — /plan, prompt, review inline diff, accept หรือ reject, รัน script จากนั้นผู้เข้าร่วมสร้างเวอร์ชันของตัวเอง

การเรียนรู้สำคัญ: /plan ในฐานะเครื่องมือกำหนดทิศทางก่อน execute, การ review การเปลี่ยนแปลงของ Claude ใน diff view ของ VS Code และการปรับ prompt เมื่อผลลัพธ์ไม่ตรงกับที่ต้องการ
ทำไมต้องเลือก project นี้: ทีม IT ทุกทีมทำ report แบบนี้ด้วยมือ มันแทนที่งานที่ทำซ้ำๆ ได้ทันที ผลิต HTML output ที่มองเห็นได้ และ /plan step เหมาะธรรมชาติมากที่นี่ — โครงสร้าง report ควรถกเถียงก่อนที่ Claude จะสร้าง Python คือ vehicle; ผู้เข้าร่วมไม่จำเป็นต้องอ่านโค้ดแม้แต่บรรทัดเดียว

Live demo — Claude Code meets the Jira / monday.com API
Demo โดยผู้สอนเท่านั้น โดยใช้ API token ที่เตรียมไว้ล่วงหน้า แสดง Claude Code ที่ดึงข้อมูลจริงจาก Jira หรือ monday.com โดยตรง — ไม่มี CSV ไม่มี copy-paste ขอให้ดึง open tickets ที่ assign ให้ผู้ใช้คนใดคนหนึ่ง หรือสรุป board items ตาม status จุดสำคัญคือความแตกต่าง: "ก่อนหน้านี้คุณส่งไฟล์ให้ Claude ตรงนี้ Claude ไปดึงข้อมูลเองเลย"

Frame ให้ชัดเจนว่านี่คือสะพานสู่ Session 3: MCP จะ automate และขยาย pattern นี้ เก็บ credentials และ config ไว้กับผู้สอน — นี่คือ demo ไม่ใช่ hands-on exercise
การเตรียมตัวของผู้สอน: เตรียม read-only API token ก่อนเซสชัน Jira: Settings → Personal Access Tokens / monday.com: Profile → Developers → My Access Tokens ทดสอบ connection ก่อน — demo ที่ fail เรื่อง auth ทำลาย momentum ได้มาก

Token efficiency — first principles
แนะนำ context window ด้วย analogy ง่ายๆ: หน่วยความจำระยะสั้นในการทำงานของ Claude Code แสดงให้เห็นว่าเกิดอะไรขึ้นเมื่อเซสชันยาวเกินไป สอน 2 นิสัย: /compact (สรุปและทำต่อ) และ /clear (เริ่มใหม่สำหรับงานใหม่) เชื่อมโยงกับค่าใช้จ่าย — ใช้ token น้อยลง = ค่า API ต่อเซสชันถูกลง สังเกตว่า panel ของ Claude Code ใน VS Code แสดงความยาวเซสชันแบบ visual ซึ่งช่วยได้

MCP — connecting Claude Code to your tools
การแนะนำแบบ demo-first สั้นๆ แสดง Claude Code ที่ดึงข้อมูล Jira ticket จริงด้วย ID โดยใช้ Atlassian MCP server — ไม่ต้องคัดลอก ไม่ต้องวาง Claude Code ทำงานโดยตรงใน Jira อธิบาย mental model: MCP คือสิ่งที่ทำให้ Claude Code สามารถ take action ภายในเครื่องมือของคุณได้ ไม่ใช่แค่สร้างข้อความให้คุณ paste

เก็บรายละเอียด configuration ไว้นอก scope ของเซสชันนี้ — frame ว่า "นี่คือสิ่งที่เป็นไปได้ นี่คือที่ที่ต้องไปต่อ" ชี้ไปที่ docs.claude.com/en/docs/claude-code สำหรับ MCP setup guide
เปรียบเทียบให้ชัดเจนกับ product map ของ Session 1: Connectors (Claude.ai) = one-click ไม่ต้องตั้งค่า / MCP (Claude Code) = ต้องตั้งค่าเอง full programmatic control — concept เดียวกัน power level ต่างกัน

Token efficiency — advanced habits
แนะนำ .claudeignore เพื่อยกเว้นโฟลเดอร์ขนาดใหญ่หรือที่ไม่เกี่ยวข้อง (attachments, exports, archives) ออกจาก context ของ Claude Code แสดงให้เห็นว่า VS Code workspace ที่ scope ดีช่วยลดการใช้ token และให้ผลลัพธ์ที่ focused มากขึ้น กลับมาที่ CLAUDE.md ในฐานะ token-efficiency tool — คำสั่งที่ชัดเจนป้องกัน Claude จากการใช้ token ไปกับการเดาเจตนาของคุณ

How to keep learning independently
